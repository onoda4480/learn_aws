# IAM Identity Center

## 概要

Organizations 配下の複数AWSアカウントへのアクセスを**一元管理**するサービス

## 主な機能

- 既存の SAML 2.0 IdP と接続
- ユーザー・グループの自動プロビジョニング（SCIM）
- 1回のサインインで複数アカウントにアクセス（SSO）
- 権限セットで各アカウントへのアクセス権を定義

## 構成図

```
既存 SAML 2.0 IdP（社内認証基盤）
       │
       ▼
IAM Identity Center（SSO ポータル）
       │
       ├─→ アカウントA（開発）  → 開発者グループに許可
       ├─→ アカウントB（本番）  → 運用チームに許可
       └─→ アカウントC（監査）  → 監査チームに許可
```

## 試験問題パターン

### シナリオ
- Organizations で複数AWSアカウント運用
- 既存 SAML 2.0 IdP がある
- 従業員が役割に応じて複数アカウントにアクセス
- 管理負担を最小限に

### 正解：IAM Identity Center

### 誤答選択肢

| 選択肢 | なぜ不正解か |
|--------|-------------|
| RAM | リソース共有のサービス。ユーザー認証機能なし |
| Secrets Manager | シークレット保管のサービス。SSO機能なし |
| Cognito | Webアプリ向け。マルチアカウント管理には不向き |
