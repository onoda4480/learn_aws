# Route 53 ルーティングポリシー

## 7種類のルーティングポリシー

| ポリシー | 用途 | ヘルスチェック |
|---------|------|---------------|
| シンプル | 1つのリソースへルーティング | 任意 |
| 加重 | 割合を指定して分散 | 任意 |
| レイテンシー | 最も遅延が少ないリージョンへ | 任意 |
| フェイルオーバー | プライマリ障害時にセカンダリへ | **必須** |
| 位置情報 | ユーザーの地理的位置に基づく | 任意 |
| 地理的近接性 | リソースとの距離に基づく | 任意 |
| 複数値回答 | 複数IPをランダムに返す（正常なものだけ） | **必須** |

## ヘルスチェックが必須なポリシー

```
フェイルオーバー：障害検知が目的 → ヘルスチェック必須
複数値回答：正常なIPだけ返す → ヘルスチェック必須
```

### 複数値回答（Multivalue Answer）の仕組み
- 最大8つのIPアドレスを返す
- ヘルスチェックで**正常なものだけ**をクライアントに返す
- ヘルスチェックなしだと「複数値回答」の意味がない

## 試験ポイント

```
「ヘルスチェックが必須」→ フェイルオーバー、複数値回答
```
